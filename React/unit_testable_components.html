Testing UI behavior is really hard, and you can get lulled into writing sloppy tests. Sloppy tests are unmaintainable and encourage "behavior locking". It's a problem that only grows as time goes on

Testing React components is really hard

Frontend testing requires, in one form or another, a mocked DOM, which is a beast in its own rite. Your tests can't be run like a basic function, because the true test behavior is browser-based. Which is what enzyme was created for. It's a mandatory supplement to jest for component testing.
